# 📚 Electronic Diary — Система управления школой

> Enterprise SaaS-платформа для автоматизации административных, учебных и коммуникационных процессов в школах. Построена на Java 17, Spring Boot 3.5.7, и оснащена AI-ассистентом на базе GPT-4o-mini.

**Стек:** Java 17 · Spring Boot 3.5.7 · Spring Security + JWT · PostgreSQL · Flyway · Thymeleaf · Docker · OpenAI GPT-4o-mini

## 🤖 AI-ассистент

AI-ассистент — одна из главных фишек системы. Он позволяет администраторам и директорам **управлять всей базой данных школы через обычный чат** — без ручного заполнения форм.

### Какую проблему он решает

Настройка школы с нуля — это больно. Администратору нужно:
1. Создать 10–20 учебных групп по одной
2. Добавить сотни учеников с ФИО, email, паролями, привязкой к группам
3. Создать 15–20 учителей и прикрепить каждого к предметам
4. Вручную составить расписание — подобрать учителей, предметы, кабинеты, временные слоты
5. Повторять всё это каждый учебный год

Через обычный UI это занимает **часы кликов по формам**. Одна ошибка — и нужно возвращаться и исправлять записи по одной.

### Как AI-ассистент это решает

Ассистент использует **OpenAI Function Calling (Tool Use)** — механизм, при котором GPT не просто отвечает текстом, а решает какую операцию с базой данных выполнить и вызывает её напрямую.

**Примеры диалогов:**

```
Администратор: "Создай классы 9А, 9Б, 10А, 10Б по 25 учеников"
AI: ✅ Группа 9А: 25 учеников
    ✅ Группа 9Б: 25 учеников
    ✅ Группа 10А: 25 учеников
    ✅ Группа 10Б: 25 учеников
```

```
Администратор: "Добавь 5 учителей математики и физики"
AI: ✅ Учителя (5) созданы: Смирнов Иван, Петров Алексей... Предметы: Математика, Физика
```

```
Администратор: "Сгенерируй расписание для группы 9А с понедельника по 6 уроков в день"
AI: ✅ Расписание для 9А создано: 30 уроков (пн-пт с 2025-01-20)
```

```
Администратор: "Кто из учеников отстаёт? Покажи средние оценки по классам"
AI: Анализ успеваемости:
    Средняя оценка по классам: 9А: 3.8, 9Б: 4.1, 10А: 3.5
    По предметам: Математика: 4.2, История: 3.6
    Троечники: Иванов Иван (3.1), Козлов Пётр (3.0)
```

### Доступные инструменты AI

| Инструмент | Кто может | Что делает |
|---|---|---|
| `get_groups` | Все | Список всех групп с количеством учеников |
| `get_students` | Все | Список учеников с фильтром по группе |
| `get_teachers` | Все | Список учителей с предметами |
| `get_disciplines` | Все | Список учебных предметов |
| `get_student_marks` | Все | Оценки конкретного ученика по предметам |
| `get_stats` | Все | Общая статистика школы |
| `get_schedule` | ADMIN/DIRECTOR | Расписание группы на ближайшую неделю |
| `add_student` | ADMIN/DIRECTOR/TEACHER | Добавить одного ученика с явно указанным ФИО |
| `add_students_random` | ADMIN/DIRECTOR | Массовое создание учеников с рандомными русскими ФИО |
| `add_group` | ADMIN/DIRECTOR | Создать одну группу |
| `add_multiple_groups` | ADMIN/DIRECTOR | Создать несколько групп сразу (+ учеников в каждую) |
| `add_teacher` | ADMIN/DIRECTOR | Создать одного или нескольких учителей, привязать к предметам |
| `add_discipline` | ADMIN/DIRECTOR | Добавить новый предмет |
| `generate_schedule` | ADMIN/DIRECTOR | Автоматически сгенерировать расписание для группы |
| `analyze_performance` | ADMIN/DIRECTOR | Анализ успеваемости: средние, отличники, троечники |
| `add_course` | ADMIN/DIRECTOR | Создать факультатив и назначить учителя |
| `delete_student` | ADMIN/DIRECTOR | Удалить ученика по имени |
| `delete_all_students` | ADMIN/DIRECTOR | Удалить учеников (с фильтром по группе/количеству) |
| `delete_all_teachers` | ADMIN/DIRECTOR | Удалить всех учителей (предварительно очищает расписание) |

### Как это работает технически

1. Пользователь пишет сообщение в чат-интерфейсе
2. Бэкенд формирует запрос к OpenAI — в системный промпт инжектируется текущее состояние БД (группы, предметы, количество учеников/учителей)
3. GPT решает какой инструмент(ы) вызвать и возвращает `tool_calls` с именем функции и аргументами
4. Бэкенд выполняет соответствующий Java-метод — данные сохраняются напрямую в PostgreSQL
5. Результат возвращается пользователю в виде отформатированного сообщения

Ассистент хранит **историю диалога для каждого пользователя** (последние 20 сообщений) — поэтому понимает контекст в уточняющих запросах.

## 🚀 Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone https://github.com/boyutov/Electronic-diary.git
cd Electronic-diary

# 2. Создать файл с переменными окружения
cp env.example .env
# Открыть .env и заполнить DB_PASSWORD, JWT_SECRET, и OPENAI_API_KEY

# 3. Запустить PostgreSQL
docker compose up -d

# 4. Собрать и запустить приложение
./mvnw spring-boot:run       # Linux / macOS
mvnw.cmd spring-boot:run     # Windows

# 5. Открыть в браузере
# Лендинг:    http://localhost:8088
# Swagger UI: http://localhost:8088/swagger-ui.html
```

```bash
# Запустить PostgreSQL в фоновом режиме
docker compose up -d

# Проверить статус
docker compose ps
```

- **Хост:** `localhost`

## 🏫 Первоначальная настройка школы

### Шаг 1 — Регистрация школы

```bash
curl -X POST http://localhost:8088/api/purchase \
  -H "Content-Type: application/json" \
  -d '{"schoolName": "SchoolA", "contactEmail": "admin@school.ru", "studentCount": 200, "months": 12}'
```

В ответе — **временный пароль активации**.

### Шаг 2 — Активация и создание первого администратора

```bash
curl -X POST http://localhost:8088/api/purchase/register \
  -H "Content-Type: application/json" \
  -d '{
    "schoolName": "SchoolA",
    "accessPassword": "ВРЕМЕННЫЙ_ПАРОЛЬ",
    "firstName": "Иван",
    "secondName": "Иванов",
    "email": "admin@school.ru",
    "password": "AdminPass123!"
  }'
```

### Шаг 3 — Вход и получение JWT-токена

```bash
curl -X POST http://localhost:8088/api/auth/authenticate \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@school.ru", "password": "AdminPass123!", "schoolName": "schoola"}'
```

Используйте полученный токен в заголовке `Authorization: Bearer <token>` для всех API-запросов.

### Шаг 4 — Открыть панель управления

Перейдите на `http://localhost:8088/schoola/admin` и войдите.

### Шаг 5 — Настройка через AI-ассистент

Нажмите на иконку чата в панели управления и попросите ассистента настроить школу:

```
"Создай предметы: математика, физика, химия, история, литература, английский"
"Создай классы 9А, 9Б, 10А, 10Б с 25 учениками в каждом"
"Добавь 8 учителей, по 2 на каждый предмет"
"Сгенерируй расписание для всех классов на эту неделю"
```

## 👥 Роли и доступ

| Роль | Описание | URL панели |
|---|---|---|
| `ADMIN` | Полное управление школой | `/{school}/admin` |
| `DIRECTOR` | Статистика, жалобы, персонал | `/{school}/director` |

## 🔧 Устранение ошибок

### `Connection refused` к PostgreSQL
```bash
docker compose ps       # проверить статус контейнера
docker compose up -d    # запустить если не работает
```

### `DB_PASSWORD` не определён
```bash
export $(cat .env | xargs) && ./mvnw spring-boot:run
```

### Flyway checksum mismatch
```sql
-- Подключиться к БД и удалить историю Flyway (только для разработки!)
DROP TABLE flyway_schema_history;
```

### Порт 8088 занят
```bash
lsof -i :8088           # найти процесс
kill -9 <PID>           # завершить его
```

### AI-ассистент не отвечает
- Убедитесь что `OPENAI_API_KEY` задан в `.env`
- Проверьте баланс на platform.openai.com
- Без ключа ассистент работает только в режиме чтения (без изменений БД)
